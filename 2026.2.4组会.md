# 在 Qwen3-VL 中缝合 Engram 的实现方案
## 1. Engram 的缝合位置设计
### 1.1 Qwen3-VL 的整体结构
Qwen3-VL 的视觉–语言处理流程可以抽象为三部分：

1. **Vision Encoder（ViT）**
    - 将输入图像/视频帧切分为 patch
    - 输出 patch-level token 表征
2. **Merger / Projector**
    - 对视觉 token 做压缩、投影
    - 将视觉特征映射到语言模型可接收的维度
3. **LLM（Decoder）**
    - 基于视觉 token + 语言 token 进行推理与生成

在该结构中，小目标信息主要存在于 **Vision Encoder 的 patch token 阶段**，一旦进入 Merger 压缩后，局部细节会进一步被弱化。

---

### 1.2 Engram 的核心缝合原则
Engram 的目标不是替代模型决策，而是**在注意力竞争开始前，对“可能包含局部异常的小区域 token”进行加权**。

因此缝合位置遵循以下原则：

+ 尽量靠前（小目标尚未被完全语义化）
+ 不破坏模型主干结构

---

### 1.3 缝合位置
**位置：Vision Encoder 输出 → Merger / Projector 之前**

即：

```latex
Video Frame
  → Vision Encoder (patch tokens)
  → Engram-based token reweighting   ← 缝合点
  → Merger / Projector
  → LLM
```


![](https://cdn.nlark.com/yuque/0/2026/png/33077450/1770180417611-a5342afa-0bb6-4edf-9a65-21b503ad7ebc.png)

#### 选择该位置的原因
+ patch token 仍保留局部空间信息
+ 不干扰 LLM 的高层语义推理能力
+ 对全局异常检测影响极小
+ 工程实现简单，可控性强

## 2. Engram 向量的获取方法
### 2.1 向量的基本定义
Engram 中存储的向量是：

> **来自 Qwen3-VL 视觉编码器中，能够代表“局部异常证据”的 patch / ROI 级视觉特征向量**
>

这些向量不是整帧特征，也不是异常结论，而是**可复用的局部视觉模式表示**。

---

### 2.2 从视觉编码器获取 patch token 隐状态
#### 2.2.1 选择合适的层
+ 使用 **Vision Encoder 的浅层或中层 hidden states**
+ 原因：
    - 浅层：保留纹理、边缘、小尺度结构
    - 深层：更偏语义，小目标容易被洗掉

经验上可选择：

+ Vision Encoder 前 1/3 ～ 1/2 层

### 2.3 从 patch token 到局部区域向量
#### 2.3.1 空间区域映射
不依赖 tracking，只使用**粗空间信息**：

+ 将图像划分为 `3×3` 或 `4×4` grid
+ 每个 grid 对应一组 patch token
+ 根据异常标注（或时间段）选取目标 grid 内的 token 集合

---

#### 2.3.2 token 筛选（去背景）
在选定区域内，对 token 做轻量筛选：

+ 去除范数极小的 token（低信息量）
+ 去除与区域均值高度相似的 token（背景一致区域）
+ 可选：与正常样本 token 对比，排除高度相似者

目的：  
保留“与周围背景差异较大”的局部特征。

### 2.4 原型去重与入库
+ 对同一异常类型的候选向量进行聚类（如 K-means / HDBSCAN）
+ 每个聚类保留中心或代表向量
+ 每类异常保留几十到数百个原型即可

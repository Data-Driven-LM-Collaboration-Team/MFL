## 1. 研究目标

本项目面向视频异常检测中的视觉 token 冗余问题，提出 **ObjEviPool：Object Evidence Pooling for Video Anomaly Detection**。核心思想不是直接剪掉 patch token，而是先恢复对象证据、形成对象组，再将对象组池化为 object token，从而在极低 token 预算下保留异常检测所需的对象状态、位置和交互信息。

## 2. 方法流程

如图所示，输入流式视频帧后，首先经过 Qwen3-VL ViT 和 LoRA 适配器，得到 dense visual tokens。随后通过 objectness head 和 grouping head，分别预测每个 token 的对象响应分数和分组嵌入。接着采用 local-global high-recall proposal：一方面使用阈值筛选高 objectness token，另一方面加入 global top-k 和 local top-k，避免远处小目标、边缘人物被漏掉。

在获得 proposal tokens 后，方法通过空间连通性和 embedding 相似性形成 object-evidence connected components。每个对象组再根据 mean objectness、max objectness、面积、运动、新颖性和交互风险等因素进行评分，并通过动态 object budget 为复杂帧分配更多 token。

最终，每个对象组内部执行 many-to-one object-constrained pooling，将多个 patch tokens 压缩成一个 object token。同时保留三类 scene anchors：uniform scene、boundary scene 和 scene-change tokens，用于补充全局布局、对象周边上下文和突发场景变化。压缩后的 object tokens 与 scene tokens 被送入 Temporal VAD Head，输出 frame-level anomaly score。

## 3. 核心优势

该方法相比普通 token pruning 的优势在于：第一，压缩单位从 patch 提升到 object，避免破坏对象完整性；第二，local-global proposal 提升小目标召回；第三，动态预算能根据场景复杂度自适应分配 token；第四，scene anchors 保留异常检测所需的场景上下文。

## 4. 当前进展与下一步

当前实验已经验证，Qwen3-VL ViT 经过 LoRA 和 token-level supervision 后，可以恢复较高召回的 objectness 信号，并将每帧约 2020 个 dense tokens 压缩到约 20–50 个 compressed tokens。下一步需要接入 VAD head，系统比较 full tokens、random、uniform、top-objectness、representative tokens、pooled object tokens 和 object+scene tokens，在 AUROC、AP、F1、latency 和 token reduction 上验证该方法的有效性。
